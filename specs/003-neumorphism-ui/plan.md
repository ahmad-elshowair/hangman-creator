# Implementation Plan: Neumorphism UI Effect

**Branch**: `003-neumorphism-ui` | **Date**: 2026-03-31 | **Spec**: [spec.md](file:///Users/elshowair/code/mine/hangman-creator/specs/003-neumorphism-ui/spec.md)

## Summary

Apply neumorphic (soft 3D) visual styling to the Hangman Creator's primary surfaces and interactive controls. This involves creating a centralized shadow token system and integrating it into MUI component overrides and inline styles across the Setup and Play pages. The effect uses moderate-depth dual shadows (6-10px blur), a shadow-flip interaction model for buttons, green-tinted edge glow for dark mode, and 200ms ease-in-out transitions throughout.

## Shadow Token Design

### Light Mode (`#F0F4F2` background)

The light mode uses the classic neumorphic dual-shadow technique. The background color `#F0F4F2` (Soft Mint) is the reference surface from which shadows are derived.

| Token | CSS Value | Purpose |
|-------|-----------|---------|
| `light.raised` | `6px 6px 12px rgba(0, 0, 0, 0.08), -6px -6px 12px rgba(255, 255, 255, 0.9)` | Default raised surface (cards, panels) |
| `light.raisedSmall` | `3px 3px 6px rgba(0, 0, 0, 0.08), -3px -3px 6px rgba(255, 255, 255, 0.9)` | Small raised elements (chips, keys) |
| `light.inset` | `inset 3px 3px 6px rgba(0, 0, 0, 0.08), inset -3px -3px 6px rgba(255, 255, 255, 0.9)` | Pressed elements (inputs, active buttons) |
| `light.flat` | `2px 2px 4px rgba(0, 0, 0, 0.06), -2px -2px 4px rgba(255, 255, 255, 0.7)` | Hover-reduced depth |

### Dark Mode (`#0F1A14` background)

Dark mode uses a green-tinted edge glow technique since classic dual shadows are invisible on dark surfaces.

| Token | CSS Value | Purpose |
|-------|-----------|---------|
| `dark.raised` | `6px 6px 12px rgba(0, 0, 0, 0.4), -3px -3px 8px rgba(102, 187, 106, 0.07)` | Default raised surface with faint emerald highlight |
| `dark.raisedSmall` | `3px 3px 6px rgba(0, 0, 0, 0.4), -2px -2px 4px rgba(102, 187, 106, 0.07)` | Small raised elements |
| `dark.inset` | `inset 3px 3px 6px rgba(0, 0, 0, 0.5), inset -2px -2px 4px rgba(102, 187, 106, 0.05)` | Pressed elements |
| `dark.flat` | `2px 2px 4px rgba(0, 0, 0, 0.3), -1px -1px 3px rgba(102, 187, 106, 0.05)` | Hover-reduced depth |

### State-Specific Tokens

| Token | CSS Value | Purpose |
|-------|-----------|---------|
| `correctInset` | `inset 2px 2px 5px rgba(46, 125, 50, 0.3), inset -2px -2px 5px rgba(102, 187, 106, 0.15)` | Correct keyboard key (green-tinted press) |
| `wrongInset` | `inset 2px 2px 5px rgba(211, 47, 47, 0.3), inset -2px -2px 5px rgba(255, 82, 82, 0.1)` | Wrong keyboard key (red-tinted press) |

### Transition Token

All neumorphic transitions use: `all 200ms ease-in-out`

## Component Override Plan

### 1. `src/constants/neumorphism.ts` [NEW]

A new constants file exporting all shadow tokens as a typed `as const` object. Structure:

```typescript
export const neuShadows = {
  light: { raised, raisedSmall, inset, flat },
  dark:  { raised, raisedSmall, inset, flat },
  correctInset,
  wrongInset,
  transition: "all 200ms ease-in-out",
} as const;
```

### 2. `src/theme.ts` [MODIFY]

| Component | Light Mode Override | Dark Mode Override |
|-----------|--------------------|--------------------|
| `MuiPaper` | Remove `border`, set `boxShadow: neuShadows.light.raised`, set `background: #F0F4F2` **(intentionally matches page background — neumorphic surfaces must share the parent background for shadows to create the depth illusion)** | Remove `border` + `backdropFilter`, set `boxShadow: neuShadows.dark.raised`, set opaque `background: #152A1C` **(shift from translucent `rgba(15, 26, 20, 0.85)` to opaque — neumorphism requires opaque surfaces for shadows to render correctly)** |
| `MuiCard` | Remove `border`, set `boxShadow: neuShadows.light.raised` | Remove `border` + `backdropFilter`, set `boxShadow: neuShadows.dark.raised` |
| `MuiButton.containedPrimary` | Replace `boxShadow` with `neuShadows.light.raisedSmall`, add `&:active` with `neuShadows.light.inset` (shadow flip) | Same pattern with dark tokens |
| `MuiTextField` | Replace `border` override with `boxShadow: neuShadows.light.inset` for the input field | Same with dark tokens |

### 3. `src/components/Keyboard.tsx` [MODIFY]

Replace the existing glass-style `backdropFilter + border` approach:
- **Default key**: `neuShadows.light.raisedSmall` / `neuShadows.dark.raisedSmall`
- **Correct guess**: `neuShadows.correctInset` with green tint
- **Wrong guess**: `neuShadows.wrongInset` with red tint and reduced opacity
- **Hover**: `neuShadows.light.flat` / `neuShadows.dark.flat` (subtle depth reduction)
- All transitions: `neuShadows.transition`
